# SurveyBox 生产环境资源要求

本文描述将 SurveyBox 部署到腾讯云（或其他云厂商）时，需要提前准备的基础设施与第三方服务。

## 一、架构概览

SurveyBox 支持两种数据库部署方式，**二选一**：

### 方案 A：Docker 内置 PostgreSQL（默认）

适合 PoC、小规模、单机一站式部署。

```
用户浏览器
    │ HTTPS
    ▼
域名 + SSL 证书
    │
    ▼
腾讯云 CVM（Docker Compose）
    ├── web（Nginx :80/443）── 静态前端 + /api 反代
    ├── api（Go :8080）────── 业务 API、文件加解密
    └── postgres（:5432）─── 仅 Docker 内网，不对外暴露
            │
            ▼
    宿主机磁盘
    ├── /data/workspace/db/pg/SurveyBox   数据库文件
    └── Docker 卷 file_data               加密附件（一期本地存储）
```

### 方案 B：独立 PostgreSQL（备选）

适合生产环境、需要自动备份/高可用、或希望计算与数据库分离的场景。CVM 上**只跑** `api` + `web`，数据库使用外部实例。

```
用户浏览器
    │ HTTPS
    ▼
域名 + SSL 证书
    │
    ▼
腾讯云 CVM（Docker Compose）
    ├── web（Nginx :80/443）
    └── api（Go :8080）
            │
            │ 内网 / 专线（VPC）
            ▼
独立 PostgreSQL
    ├── 腾讯云 TencentDB for PostgreSQL（推荐）
    ├── 另一台 CVM 自建 PostgreSQL
    └── 其他云厂商托管 PG（需网络互通）

CVM 宿主机磁盘
    └── Docker 卷 file_data               加密附件
```

可选（二期）：
    ├── 腾讯云 COS ── 附件对象存储
    └── 邮件服务 ─── 分享通知（暂未实现）

## 二、必需资源

### 1. 云服务器（CVM）

SurveyBox 应用层采用 **Docker Compose** 部署。默认方案中 `postgres`、`api`、`web` 三个容器同机运行；选用**独立 PostgreSQL** 时，CVM 仅运行 `api` + `web`。

| 项目 | 方案 A（内置 PG） | 方案 B（独立 PG） |
|------|-------------------|-------------------|
| 实例规格 | 2 核 4 GB 起 | 2 核 4 GB 起（无本地 PG，资源压力更小） |
| 系统盘 | 50 GB SSD 起 | 50 GB SSD 起 |
| 数据盘 | 100 GB+（库 + 附件） | 50 GB+（主要留给附件） |
| 操作系统 | Ubuntu 22.04 / Debian 12 | 同左 |
| 公网带宽 | 5 Mbps 起 | 5 Mbps 起 |

小规模（&lt; 50 并发填写）以上规格即可；问卷量大、附件多建议 4 核 8 GB。需 Docker Engine 24+ 与 Docker Compose V2。

**CVM 上需安装：**

- Docker Engine
- Docker Compose V2
- Git（拉取代码）
- 可选：`curl`、`openssl`（生成 `JWT_SECRET`）

**磁盘目录规划（推荐）：**

| 路径 | 方案 A | 方案 B |
|------|--------|--------|
| `/data/workspace/db/pg/SurveyBox` | PostgreSQL 数据（`POSTGRES_DATA_DIR`） | 不需要 |
| `/data/workspace/app/SurveyBox` | 应用代码与 `.env` | 同左 |
| Docker 卷 `file_data` | 答卷附件（AES-256-GCM 加密） | 同左 |

> 方案 A：数据库与附件建议放在**独立数据盘**。  
> 方案 B：数据库备份由托管实例或独立 PG 服务器负责，应用 CVM 重点保障附件磁盘容量。

---

### 1.1 独立 PostgreSQL（方案 B 备选）

当不使用 Docker 内置 `postgres` 容器时，需另行准备 PostgreSQL **15+** 实例（与项目迁移脚本兼容）。

#### 可选形态

| 形态 | 说明 | 适用场景 |
|------|------|----------|
| **腾讯云 TencentDB for PostgreSQL** | 托管服务，自动备份、监控、主从可选 | 生产推荐 |
| **独立 CVM 自建 PostgreSQL** | 自控度高，运维成本自担 | 有 DBA 或预算有限 |
| **其他云托管 PG** | 需与 CVM 网络互通、延迟可接受 | 多云 / 混合云 |

#### 规格建议（TencentDB / 自建参考）

| 项目 | 小规模 | 中等规模 |
|------|--------|----------|
| 版本 | PostgreSQL 15+ | PostgreSQL 15+ |
| 规格 | 1 核 2 GB 起 | 2 核 4 GB+ |
| 存储 | 50 GB SSD 起 | 100 GB+，支持自动扩容 |
| 地域 | **与 CVM 同地域** | 同左，降低延迟与内网流量费 |
| 网络 | 与 CVM 同一 VPC | 同左 |

#### 网络与安全

| 项目 | 要求 |
|------|------|
| 访问方式 | 优先 **VPC 内网地址**，不要对公网开放 5432 |
| 安全组 | 仅允许应用 CVM 的内网 IP 访问数据库 5432 |
| 白名单 | TencentDB 控制台配置 CVM 子网或安全组 |
| SSL | 生产建议开启；连接串加 `sslmode=require`（视实例配置） |

#### 应用配置

1. 在数据库中创建库与用户（或使用控制台初始化）：
   - 数据库名：如 `surveybox`
   - 用户：如 `surveybox`，授予该库读写权限

2. 在应用 CVM 的 `.env` 中设置完整 `DATABASE_URL`（使用数据库**内网地址**），例如：

```env
DATABASE_URL=postgres://surveybox:你的密码@10.0.1.12:5432/surveybox?sslmode=disable
POSTGRES_USER=surveybox
POSTGRES_PASSWORD=你的密码
POSTGRES_DB=surveybox
```

3. 新增 `docker-compose.override.yml`（勿提交密钥），覆盖 compose 内默认的 `@postgres:5432` 连接：

```yaml
services:
  api:
    environment:
      DATABASE_URL: ${DATABASE_URL}
```

4. 仅启动应用容器：

```bash
docker compose up -d api web
```

> 默认 `docker-compose.yml` 会将 `api` 的 `DATABASE_URL` 写死为容器名 `postgres`；使用独立 PG 时必须通过 override 或修改 compose，让 api 读取 `.env` 中的外部地址。

5. 首次连接时 API 会自动执行迁移并创建管理员账号（与内置 PG 行为一致）。

#### 方案对比

| 维度 | 方案 A：Docker 内置 PG | 方案 B：独立 PG |
|------|------------------------|-----------------|
| 部署复杂度 | 低，一键 `deploy.sh` | 中，需先建库与网络 |
| 运维 | 自管备份、升级 | 托管实例省运维（TencentDB） |
| 高可用 | 需自行做主从 / 快照 | TencentDB 可选主从、跨 AZ |
| 资源隔离 | DB 与应用争抢 CVM 资源 | 计算与存储分离 |
| 成本 | 无额外数据库费用 | 增加 TencentDB 月租 |
| 适用 | 开发、试点、小团队 | 生产、合规、持续增长 |

### 2. 域名

用于管理端访问、问卷分享链接生成（`WEB_ORIGIN`）。

| 项目 | 要求 |
|------|------|
| 域名 | 已注册并可解析，如 `survey.example.com` |
| DNS | A 记录指向 CVM 公网 IP |
| 配置 | `.env` 中设置 `WEB_ORIGIN=https://survey.example.com` |

分享链接格式为 `{WEB_ORIGIN}/f/{token}`，域名错误会导致受访者无法打开问卷。

**中国大陆上线**：若使用国内 CVM + 自定义域名对外提供 Web 服务，通常需要完成 **ICP 备案**（以当地监管要求为准）。

---

### 3. SSL 证书（HTTPS）

生产环境**必须**启用 HTTPS，否则：

- 登录 Cookie（`httpOnly`）在 HTTP 下存在被窃听风险
- 现代浏览器对非 HTTPS 站点限制增多

| 方案 | 说明 |
|------|------|
| 腾讯云 SSL 证书 | 可免费申请 DV 证书，或购买 OV/EV |
| Let's Encrypt | 免费，可用 Certbot + Nginx 自动续期 |
| 负载均衡 CLB 证书 | 若前端使用 CLB 终结 HTTPS，证书挂在 CLB 上 |

当前 Docker 内置 Nginx 监听 **80 端口**。生产环境常见做法：

1. **CVM 本机 Nginx/Caddy** 做 443 终结，反代到 `web:80`；或
2. **腾讯云 CLB** 挂载证书，后端转发到 CVM:80。

---

### 4. 安全组 / 防火墙

**应用 CVM**

| 端口 | 方向 | 用途 | 是否对外开放 |
|------|------|------|----------------|
| 22 | 入站 | SSH 运维 | 建议仅办公网 IP |
| 80 | 入站 | HTTP（可重定向到 HTTPS） | 是 |
| 443 | 入站 | HTTPS | 是 |
| 8080 | 入站 | API | **否**（经 Nginx 反代） |

**PostgreSQL（方案 A：Docker 内置）**

| 端口 | 说明 |
|------|------|
| 5432 | **不对公网开放**；仅 Docker 内网 `api` → `postgres` |

**PostgreSQL（方案 B：独立 / TencentDB）**

| 端口 | 说明 |
|------|------|
| 5432 | **不对公网开放**；安全组仅放行应用 CVM 内网 IP → 数据库实例 |

---

### 5. 密钥与账号（应用层，非云资源但部署前必配）

在 `.env` 中配置，**不可使用默认值上线**：

| 变量 | 说明 |
|------|------|
| `JWT_SECRET` | 登录 Token 签名密钥，建议 `openssl rand -base64 32` 生成 |
| `POSTGRES_PASSWORD` | 数据库密码 |
| `ADMIN_PASSWORD` | 管理员初始密码（仅首次建库生效） |
| `FILE_ENCRYPTION_KEY` | 附件加密密钥（可选，留空则从 JWT_SECRET 派生） |

## 三、按功能需要的可选 / 推荐资源

### 6. 大模型 API（LLM）— 使用 AI 生成问卷时必需

系统通过 **OpenAI Chat Completions 兼容协议** 调用大模型。

| 变量 | 说明 |
|------|------|
| `LLM_BASE_URL` | API 地址，如 `https://api.openai.com/v1` |
| `LLM_API_KEY` | API 密钥 |
| `LLM_MODEL` | 模型名，默认 `gpt-4o-mini` |

可选用：

- OpenAI / Azure OpenAI
- 国内兼容网关（如各类聚合 API）
- 自建推理服务（需兼容 OpenAI 格式）

**不配置 `LLM_API_KEY` 时**：手工编辑问卷、发布、收集答卷仍可正常使用，仅「AI 生成问卷」功能不可用。

---

### 7. 腾讯云 COS（对象存储）— 二期推荐

| 状态 | 说明 |
|------|------|
| 当前 | 附件存储在 CVM 本地 Docker 卷，AES 加密 |
| 规划 | 切换 `STORAGE_BACKEND=cos`（代码尚未实现） |

**何时需要 COS：**

- 附件量大，CVM 磁盘不足
- 需要跨机容灾、多副本
- 希望与 CVM 解耦，便于横向扩展

**若提前准备 COS，建议开通：**

| 项目 | 说明 |
|------|------|
| COS 存储桶 | 与 CVM 同地域，降低延迟与流量费 |
| 访问密钥 | SecretId / SecretKey（仅服务端使用，勿提交代码库） |
| 权限 | 私有读写；通过 API 预签名或后端代理下载 |
| 生命周期 | 可按需配置冷存储、过期清理 |

---

### 8. 邮件服务 — 二期（分享通知，暂未实现）

批量分享目前仅生成链接，**不会自动发邮件**。若后续实现邮件通知，需准备：

| 方案 | 说明 |
|------|------|
| 腾讯云邮件推送（SES） | 需域名验证、发信配额 |
| 企业邮箱 SMTP | 配置 SMTP 主机、端口、账号 |
| 第三方（SendGrid 等） | 需 API Key |

## 四、运维与可靠性（强烈建议）

### 9. 备份

| 对象 | 方案 A（内置 PG） | 方案 B（独立 PG） |
|------|-------------------|-------------------|
| PostgreSQL | `pg_dump` 或云硬盘快照，建议每日 | TencentDB 自动备份，或自建实例 `pg_dump` |
| 附件目录 | 同步至 COS / 快照，建议每日 | 同左 |
| `.env` | 加密离线保存，变更时备份 | 同左 |

### 10. 监控与告警

| 项目 | 说明 |
|------|------|
| 腾讯云云监控 | CPU、内存、磁盘、带宽 |
| 进程/容器 | `docker compose ps`、健康检查 `/health` |
| 磁盘水位 | 数据库 + 附件增长需重点关注 |

### 11. 日志

```bash
docker compose logs -f api
docker compose logs -f web
```

生产环境可接入 CLS（日志服务）或本地 `logrotate`。

## 七、部署前检查表

**通用**

- [ ] CVM 已安装 Docker / Docker Compose
- [ ] 域名 DNS 已指向 CVM
- [ ] HTTPS 已配置（443 可访问）
- [ ] 安全组已关闭 8080 公网访问
- [ ] `.env` 中 `JWT_SECRET`、`POSTGRES_PASSWORD`、`ADMIN_PASSWORD` 已修改
- [ ] `WEB_ORIGIN` 已设为 `https://你的域名`
- [ ] （可选）`LLM_API_KEY` 已配置
- [ ] 备份策略已确定

**方案 A（内置 PostgreSQL）**

- [ ] 数据盘已挂载，`POSTGRES_DATA_DIR` 目录可写
- [ ] 安全组未对公网开放 5432
- [ ] `./scripts/deploy.sh` 或 `docker compose up -d` 启动全套服务

**方案 B（独立 PostgreSQL）**

- [ ] TencentDB / 自建 PG 已创建，版本 ≥ 15
- [ ] 数据库与 CVM **同 VPC**，使用内网地址
- [ ] 数据库安全组仅放行应用 CVM
- [ ] `.env` 中 `DATABASE_URL` 指向外部实例
- [ ] `docker compose up -d api web`（不启动内置 postgres）
- [ ] 从 CVM 测试连通：`psql` 或 `nc -zv <内网IP> 5432`

相关部署步骤见项目根目录 [README.md](../README.md) 与 [scripts/deploy.sh](../scripts/deploy.sh)。
